GW CS Senior Design 2016

Elliot Liskin: Class Extractor

Class Extractor is an OS X application that uses natural language processing to determine what the most important concepts in a class are. All the student needs is an audio recording of the professor lecturing. Students can then use this information to organize and prioritize their studying.

Important concepts are conveyed to the user using two interfaces: Word Cloud and Timeline. Word Cloud is a series of circles whose sizes are directly correlated to the importance of their represented topics. Topic importance is also portrayed by the color of the cloud; more important topics are a deep red, with successively less important concepts becoming a more royal purple.

Timeline is the second interface. Timeline shows a temporal overview of topic discussion throughout a single lecture. Armed with this interface, students can see how different topics overlap and interrelate.

Documentation:

    This app was built using Xcode 7.3 on OS X 10.11.4.

    To run the app, simply double click on the application icon (or run it from Xcode), and click "Select Audio File". This button opens up a file picker that allows you to navigate to and pick the audio file you want to analyze. This file has to be of an audio type (most of my testing was done with mp3s, but other standard audio types will work fine as well). Before selecting "Open," make sure that you are connected to the internet and have a solid connection; once that button is pressed, Class Extractor will send the file to IBM's Watson for speech-to-text transliteration. If running the app from Xcode, the user will have to create a trial account on IBM's BlueMix website and obtain a set of keys to allow Class Extractor to access Watson's API.

    Once the relevant data has been received and Class Extractor has analyzed it, Class Extractor will present the data using the two interfaces described above. You can switch between the interfaces using the segmented control at the bottom of the window.
